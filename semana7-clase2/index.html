Concepto Fundamental

¿Qué es una variable binaria?

Definición

Una variable binaria (también llamada variable 0-1 o booleana) es una variable de decisión que solo puede tomar dos valores:

xi = { 0 si NO se elige la opción i  |  1 si SÍ se elige la opción i }

Se usan cuando las decisiones son de tipo "todo o nada": invertir o no invertir, abrir o no abrir, asignar o no asignar. No puedes hacer "medio proyecto" ni abrir "media tienda".

¿Cuándo se usan variables binarias?

  • Selección de proyectos: ¿En cuáles invertimos dado un presupuesto limitado?
  • Apertura de locales: ¿Cuáles sedes abrimos con restricciones de recursos?
  • Asignación de tareas: ¿Quién hace qué tarea para minimizar costos o tiempos?
  • Restricciones lógicas: "Si elegimos A, debemos elegir B", "No se pueden elegir A y B juntos"
💡

Diferencia con PL clásica: En Programación Lineal normal, las variables pueden tomar cualquier valor continuo (3.5 unidades, 12.7 horas). Con variables binarias, la respuesta es solo 0 o 1. Esto cambia cómo se configura Solver.

Paso a Paso

Configuración en Excel Solver

Antes de ver los ejemplos, aprendamos cómo se arma un modelo binario en Excel paso a paso.

Framework estándar en Excel

Para que siempre sepas cómo armar cualquier modelo, usaremos este formato único:

  • Una columna por variable (x₁, x₂, x₃, ...)
  • Tres columnas fijas al final: LI (lado izquierdo = fórmula), Signo (≤, ≥, =), LD (lado derecho = valor límite)
  • Fila de Valor: las celdas 0/1 que Solver cambia
  • Fila de F. Objetivo: coeficientes de Z y el resultado calculado
  • Una fila por restricción: coeficientes, LI calculado, signo y límite
ABCDEFG
1 Variables x₁x₂x₃ LI Signo LD
2 Valor 000 ← Solver cambia B2:D2
3 F. Objetivo c₁c₂c₃ =SUMAPRODUCTO(B2:D2, B3:D3) MAX/MIN
4 Restricción 1 a₁a₂a₃ =SUMAPRODUCTO(B2:D2, B4:D4) ≤ b₁
5 Restricción 2 a₁a₂a₃ =SUMAPRODUCTO(B2:D2, B5:D5) ≥ b₂
🎯

En Solver, siempre conectas la columna LI con el signo y el LD. Ejemplo: $E$4 ≤ $G$4. Las celdas de decisión son la fila de Valor.

Pasos para configurar Solver

1
Armar la tabla en Excel

Colocar los datos del problema: nombres, coeficientes del objetivo y coeficientes de cada restricción.

2
Crear las celdas de decisión

Una fila donde cada celda representa una opción. Inicialmente todas en 0. Estas son las que Solver va a cambiar.

3
Escribir la función objetivo

Usar =SUMAPRODUCTO(decisiones, coeficientes) para calcular el valor total.

4
Escribir las restricciones (lado izquierdo)

Cada restricción tiene su propio =SUMAPRODUCTO(decisiones, coeficientes_restricción).

5
Abrir Solver

Menú Datos → Solver. Si no aparece: Archivo → Opciones → Complementos → Complementos de Excel → Solver.

6
Configurar Solver

Objetivo: celda de la función objetivo.
Cambiar celdas: rango de celdas de decisión.
Restricciones: agregar las restricciones del modelo.

7
Agregar restricción BINARIA

Click en "Agregar" → seleccionar las celdas de decisión → en el operador elegir "bin" (binario). Esto restringe las variables a 0 o 1.

8
Método de resolución

Seleccionar "Simplex LP" (si todo es lineal) y hacer clic en Resolver.

⚠️

Error común: Olvidar agregar la restricción binaria. Si no la agregas, Solver tratará las variables como continuas y podrá dar resultados como 0.73 (seleccionar 73% de un proyecto), lo cual no tiene sentido.

Selección de Inversiones

Problema

Un emprendedor de Pereira tiene $80 millones COP y quiere elegir entre 5 proyectos de inversión para maximizar la ganancia total. Cada proyecto se ejecuta completo o no se hace.

ProyectoInversión (M COP)Ganancia esperada (M COP)
1. App de domicilios2540
2. Café especial3048
3. Food truck1520
4. Tienda online2030
5. Coworking3552

Paso 1 — Formulación matemática

Variables de decisión: xi = 1 si se invierte en el proyecto i, 0 si no.

Maximizar Z = 40x₁ + 48x₂ + 20x₃ + 30x₄ + 52x₅

Sujeto a:

25x₁ + 30x₂ + 15x₃ + 20x₄ + 35x₅ ≤ 80
x₁, x₂, x₃, x₄, x₅ ∈ {0, 1}

Paso 2 — Armado en Excel (framework estándar)

1 Variables x₁ Appx₂ Caféx₃ Foodx₄ Tiendax₅ Cowork LISignoLD
2 Valor 00000 ← Solver cambia B2:F2
3 F. Objetivo 4048203052 =SUMAPRODUCTO(B2:F2, B3:F3) MAX
4 Presupuesto 2530152035 =SUMAPRODUCTO(B2:F2, B4:F4) ≤ 80

Paso 3 — Configuración de Solver

1
Establecer objetivo:

Celda G3 (LI de F. Objetivo) → Máximo

2
Cambiando las celdas:

Rango de decisión: B2:F2 (fila de Valor)

3
Restricción 1 — Presupuesto:

G4 ≤ I4 (LI de la fila 4 ≤ LD de la fila 4)

4
Restricción 2 — Binarias:

B2:F2 → operador "bin"

5
Método: Simplex LP → Resolver

Invertir en: App de domicilios ($25M), Tienda online ($20M) y Coworking ($35M).

Ganancia total: 40 + 30 + 52 = $122M.

💡

¿Por qué no el Café? Aunque el café da $48M de ganancia, requiere $30M. Combinando App + Tienda + Coworking se usan los $80M de forma más eficiente ($122M total vs. cualquier otra combinación factible).

Asignación de Tareas

Problema

Una empresa de logística en Pereira tiene 3 repartidores y 3 zonas de entrega. Cada repartidor tiene un tiempo diferente según la zona (en minutos por ruta). Se quiere minimizar el tiempo total asignando exactamente un repartidor a cada zona.

RepartidorZona CentroZona CubaZona Dosquebradas
Juan253545
María302040
Pedro403025

Formulación

Variables: xij = 1 si el repartidor i se asigna a la zona j, 0 si no.

Min Z = 25x₁₁ + 35x₁₂ + 45x₁₃ + 30x₂₁ + 20x₂₂ + 40x₂₃ + 40x₃₁ + 30x₃₂ + 25x₃₃

Sujeto a:

Cada repartidor va a exactamente 1 zona (suma fila = 1)
Cada zona recibe exactamente 1 repartidor (suma columna = 1)
xij ∈ {0, 1}

Filas (repartidores): x₁₁+x₁₂+x₁₃ = 1, x₂₁+x₂₂+x₂₃ = 1, x₃₁+x₃₂+x₃₃ = 1

Selección con Restricciones Lógicas

Problema

Una empresa de Pereira quiere elegir campañas de marketing para maximizar el impacto con un presupuesto de $40 millones. Pero hay condiciones especiales:

CampañaCosto (M COP)Impacto (puntos)
1. Redes sociales825
2. Radio local1230
3. Vallas publicitarias1535
4. Influencers1028
5. Evento presencial2045
6. Email marketing515

Restricciones lógicas:

  • Si se elige Redes sociales (1), se debe elegir también Influencers (4).
  • Se deben elegir al menos 2 campañas.
  • No se pueden hacer Vallas (3) y Evento (5) al mismo tiempo (compiten por el mismo equipo).

Formulación — Cómo traducir restricciones lógicas

🔑

"Si A, entonces B" se traduce como: x₁ ≤ x₄ (o equivalente: x₁ − x₄ ≤ 0). Así, si x₁=1, entonces x₄ debe ser 1.

🔑

"Al menos N opciones" se traduce como: x₁+x₂+...+xₙ ≥ N

🔑

"No ambas a la vez" (mutuamente excluyentes) se traduce como: x₃ + x₅ ≤ 1

Maximizar Z = 25x₁ + 30x₂ + 35x₃ + 28x₄ + 45x₅ + 15x₆

Sujeto a:

8x₁ + 12x₂ + 15x₃ + 10x₄ + 20x₅ + 5x₆ ≤ 40   (presupuesto)
x₁ − x₄ ≤ 0   (si redes, entonces influencers)
x₁ + x₂ + x₃ + x₄ + x₅ + x₆ ≥ 2   (mínimo 2)
x₃ + x₅ ≤ 1   (no ambas)
x₁,...,x₆ ∈ {0, 1}

Armado en Excel (framework estándar)

Truco: para las restricciones lógicas, escribimos los coeficientes correspondientes. Por ejemplo, "x₁ − x₄ ≤ 0" se escribe como coeficientes 1, 0, 0, −1, 0, 0.

ABCDEFGHIJ
1Variables x₁ Redesx₂ Radiox₃ Vallasx₄ Influ.x₅ Eventox₆ Email LISigLD
2Valor 000000 ← Solver B2:G2
3F. Objetivo 253035284515 =SUMAPRODUCTO(B2:G2, B3:G3) MAX
4Presupuesto 8121510205 =SUMAPRODUCTO(B2:G2, B4:G4) ≤ 40
5Redes→Influ. 100−100 =SUMAPRODUCTO(B2:G2, B5:G5) ≤ 0
6Mínimo 2 111111 =SUMAPRODUCTO(B2:G2, B6:G6) ≥ 2
7No Vallas+Ev. 001010 =SUMAPRODUCTO(B2:G2, B7:G7) ≤ 1
💡

Así funcionan los coeficientes: En la fila 5, los coeficientes 1, 0, 0, −1, 0, 0 producen x₁ − x₄ al multiplicar con la fila Valor. Si se elige Redes (x₁=1) pero no Influencers (x₄=0), resulta 1 − 0 = 1, que viola la restricción ≤ 0. Para cumplirla, si x₁=1 entonces x₄ debe ser 1.

Resultado

RedesRadioVallasInfluenc.EventoEmail
Decisión 101101

Impacto máximo

103 pts

Costo total

$38M

Campañas elegidas: Redes ($8M), Vallas ($15M), Influencers ($10M) y Email ($5M).

Costo: 8 + 15 + 10 + 5 = $38M ≤ 40M.

Impacto: 25 + 35 + 28 + 15 = 103 puntos.

Verificación de restricciones lógicas:

  • Si Redes (x₁=1) → Influencers (x₄=1): 1 − 1 = 0 ≤ 0 ✓
  • Mínimo 2 campañas: 1+0+1+1+0+1 = 4 ≥ 2 ✓
  • No Vallas+Evento juntos: 1 + 0 = 1 ≤ 1 ✓

Resumen: Traducción de restricciones lógicas

Condición en españolRestricción matemática
Si A, entonces BxA ≤ xB
A y B siempre juntasxA = xB
No ambas a la vezxA + xB ≤ 1
Al menos una de las dosxA + xB ≥ 1
Exactamente N de un grupox₁ + x₂ + ... + xₖ = N
Como máximo N de un grupox₁ + x₂ + ... + xₖ ≤ N
Si A, entonces NO BxA + xB ≤ 1

Ejercicio 4

Una empresa quiere elegir programas de capacitación para maximizar el aumento de productividad con un presupuesto de $15 millones.

ProgramaCosto (M COP)Productividad (+%)
1. Excel avanzado38
2. Liderazgo512
3. Inglés715
4. Scrum410

Restricciones lógicas:

  • Si eligen Scrum (4), deben elegir Liderazgo (2): x₄ ≤ x₂
  • Deben elegir al menos 2 programas.
  • No pueden elegir Excel (1) e Inglés (3) juntos: x₁ + x₃ ≤ 1

Claves para recordar

Checklist para problemas binarios con Solver

  1. Identificar las opciones (cada una es una variable 0-1).
  2. Definir qué se quiere maximizar o minimizar (función objetivo).
  3. Traducir las restricciones del problema a desigualdades.
  4. Armar la tabla en Excel con SUMAPRODUCTO.
  5. En Solver: agregar la restricción "bin" a las celdas de decisión.
  6. Usar Simplex LP y resolver.
  7. Verificar que la solución cumple todas las restricciones.
